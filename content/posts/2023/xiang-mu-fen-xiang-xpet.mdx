# 为什么是Xpet？
因为简单！
我刚开始玩Xpet的时候脑子里想到了一个很多人记忆里的产品：QQ宠物。以QQ社交为载体，将宠物的玩法开发到了极致，虽然现在已经落幕，但其曾经的辉煌值得铭记。
Xpet选择以推特为社交载体，以插件形式提供游戏服务，同时在玩家的推特profile会有宠物动态展示。**如果能处理好跟推特的合作**，Xpet会有病毒式传播的潜力。

Xpet在arb链上，是双币系统：xpet和bpet，xpet总量3亿，bpet无上限。
因为是初期，目前只是靠宠物7级之后开始挖蓝莓，由工厂转化为bpet，也就是游戏内代币。宠物和工厂可以升级，后面只能由bpet进行升级。游戏的另一个代币xpet的玩法后面会更新，之前是可以升级用的，目前没有新的玩法。bpet支持1:1转化为xpet，但每天有限制，具体限制请关注官方说明。每个人能够挖到的蓝莓总量是有限的，总量由玩家投资的bpet数量，蓝莓挖出和蓝莓转化为bpet都有速度限制，速度跟宠物和工厂等级有关。

# 我的一些体会
1、最新更新的游戏规则很明确，能挖到的蓝莓总量由玩家充值的总量决定，只是释放速度不同（由宠物和工厂等级决定），但最终都会全部释放。如果一开始就是挖提买的思路，很难进入后面的玩法里。
2、一旦玩家挖出的蓝莓达到充值的bpet数量就会开启冒险，冒险场景里玩家可以赚取蓝莓或者bpet。
3、游戏限制蓝莓挖出总量，想要获得更多蓝莓就需要参与冒险以及后续更新的游戏场景，所以目前来看，宠物的等级重要性大于工厂，优先升级宠物。
4、宠物等级决定蓝莓的挖出速度，工厂等级决定蓝莓转化为bpet的速度。
5、宠物等级会是冒险等后续玩法的重点，大概率后面还会推出各种道具。
6、蓝莓的另一个获取途径还可以通过邀请别人来获取，需要被邀请人用非系统赠送的bpet消费才可以获得蓝莓返佣。

# 项目跟进
**`2023/12/13`**
**对于最新的规则，我的理解：**
1、玩家充值的资金会随挖矿缓慢释放，很大程度上限制了氪金玩家的过度抢跑，减少套利的可能，有利于游戏的持续性发展。
2、宠物是NFT，后期可以进行交易，这种场景玩过游戏的都不会太陌生，所以你的宠物是你最大的资产。
3、gamefi，有人会只买币不玩游戏，会对币价进行博弈，提供流动性。

**`2023/12/23`**
**xpet的逻辑简单点讲就是：**
你消费（升级宠物工厂、买宠物买道具等场景）多少bpet就会挖出多少蓝莓，蓝莓可以1:1转化成bpet，但有速度限制。
举个例子，小明刚开始玩xpet，买了1000个bpet，他用这些bpet将宠物升到了13级、工厂升到了2级，这时候他还剩41个bpet，无法进行其它操作了。宠物开始挖蓝莓，工厂开始把蓝莓转化为bpet，等他把全部挖出来的蓝莓转化为bpet后，他又有了1000个bpet，他又将宠物升到了16级，这时候他还剩133个bpet，无法进行其它操作······如此往复，当他挖出的蓝莓转化的bpet的数量（加上余额为1000个）无法支付的起任何消费场景的时候，他有三个选择：维持当前等级继续打金、卖掉bpet或者继续充值bpet。
从上面的例子可以看出，小明初始消费的bpet总量会一直进行循环产出，相当于质押，而收益就是宠物、工厂的等级提升，宠物的等级会影响到后面PVE、PVP的打金收益，所以实际的质押收益目前是宠物在PVE、PVP中带来的bpet收益。

**`2023/12/29`**
**关于bpet的通胀：**
假设初始充值10000bpet并消费，那么84%会直接打到销毁合约，剩下16%就是这部分通胀的值。除了官方的通胀部分，还有邀请的上家也会得到奖励，这部分我自己统计的是差不多25%。官方目前已经明确，只有充值的bpet进行消费才会给上家带来邀请奖励，如果复投是没有的。
由上可知，消费（充值+复投）的部分是一直有16%的通胀，且每次消费会重置返还周期。举例，小明初始充值并消费10000bpet，会分25天返还，每天400bpet。他在返还了10天之后进行了升级消耗了2000bpet，这个时候返还周期重置：（10000-400*10+2000）/25=320bpet/天。
邀请奖励也是缓慢释放的，这种机制极大的限制了通胀速度，确实会玩。

**`2024/1/9-14`**
**bpet通胀模型**
消费增发：16%，循环，缓慢释放。
boost增发：5%，一次性，缓慢释放。
假设开局bpet总量为x，增发按产出周期算，一个周期为25天，周期序号为n，第n个周期bpet总量为y(n)。每个周期消费总量占比为a，不计算周期里新进资金即不计算boost。
y(1) = x + a*x*(16%+5%) = x* (1 + a*(16%+5%))
y(2) = y(1)*(1 + a*16%)
y(3) = y(2)*(1 + a*16%) = y(1)*(1 + a*16%)2（平方）
...
y(n) = y(1)*(1 + a*16%)(n - 1)（n-1次方）
理想最大通胀：
a=100%
y(n) = x*1.21*1.16(n - 1)
y(10) = 4.6x
实际情况是：
每次消费会重置周期，实际的周期会远大于25天，此项会导致模型复杂，所以不考虑。
bpet价格上涨会导致换手率增加，假设换手的部分全部投入新进消费，上面的模型就会变为：
y(n) = y(1)*(1 + a*16% + (1 - a)*5%)n （次方）
更新后的通胀：
a=50%
y(n) = x*(1.105)n
y(10) = 2.7x
你大概可以按照上面的模型来理解bpet的通胀，事实就是不同于大家印象中的gamefi双币系统，bpet的增发量不大且很缓慢，挖提卖在这里是行不通的。
注：16%里由1%是团队的，这部分大概率是不会参与消费，但为了方便也计算了。
